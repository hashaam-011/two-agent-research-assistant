6. Critical Bugs

#1
RUN_STARTED / RUN_FINISHED missing thread_id
Spec requires {thread_id, run_id}. Backend only sent {run_id}. CopilotKit and other clients would reject or lose thread context.
Fixed
#2
STATE_DELTA payload was a snapshot object, not RFC 6902 JSON Patch
Spec: delta: List[Any]. Backend sent {active_agent, step, tool_calls}. Renamed to STATE_SNAPSHOT with correct snapshot wrapper.
Fixed
#3
TEXT_MESSAGE_CONTENT used content instead of delta; missing message_id
Standard AG-UI consumers would render empty bubbles. Renamed to delta and message_id added to all three TEXT_MESSAGE events.
Fixed
#4
TOOL_CALL_START used tool_name and stuffed args in wrong event
Spec: tool_call_name in START; args in a separate TOOL_CALL_ARGS event with delta: string. Both fixed.
Fixed
#5
TOOL_CALL_END carried result instead of using TOOL_CALL_RESULT
Spec: END carries only tool_call_id; result belongs in TOOL_CALL_RESULT with {message_id, tool_call_id, content: string, role: "tool"}. Both events now correct.
Fixed
#6
Non-spec ERROR event; LLM errors leaked into message content
Backend emitted ERROR and LLM failures were yielded as visible message text. Now emits RUN_ERROR (terminal per spec); LLM errors raised to correct handler.
Fixed
#7
Agent card: streaming: false, missing fields, wrong canonical path
Fixed: streaming: true, added defaultInputModes/defaultOutputModes, tags[] on skills, both /.well-known/agent.json and /agent-card.json now served.
Fixed
#8
A2A request/response shape is not JSON-RPC 2.0
Planner sends plain {task_id, skill, query}. Spec requires JSON-RPC 2.0 with method: "message/send" and params.message.parts[]. Requires architectural rewrite using a2a-sdk.
Open
#9
MCP server on deprecated SSE transport
transport="sse" deprecated in MCP 1.7+. Switch to transport="http". Client URL /sse → /mcp. Also: new Client per request should be a long-lived singleton.
Open
#10
web_search returns hardcoded mock results
All queries return example.com URLs. README implies real search. Integrate Tavily, Brave, or Serper to deliver on the research-assistant premise.
Open

8. Verdict & Production Roadmap

PROTOTYPE-ONLY. Excellent visual demo and well-engineered frontend. AG-UI fidelity substantially improved by this audit (7 blockers fixed). A2A and MCP remain non-conformant and require architectural investment.
Top 5 tasks before production
1
Use ag_ui.core Pydantic models as the serialization source
Serialize all events with .model_dump_json(by_alias=True) to guarantee schema compliance and make future spec upgrades automatic.
2
Rewrite planner → search as JSON-RPC 2.0 A2A
Use a2a-sdk Python package. method: "message/send", message.parts[] envelope, Task object response with artifacts.
3
Switch MCP to Streamable HTTP transport
mcp.run(transport="http"), client URL to /mcp. Reuse a singleton Client in the search agent.
4
Integrate a real web search provider
Tavily, Brave Search, or Serper. The live demo currently returns hardcoded example.com URLs — the "research assistant" premise is undermined.
5
Replace custom SSE hook with useCoAgent
Once #1 lands, use-agent-stream.ts (~145 lines) can be deleted. Remove the cosmetic CopilotKit pill or actually adopt the SDK.
